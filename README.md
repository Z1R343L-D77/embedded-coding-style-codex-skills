# Embedded Coding Style Codex Skill

> [!IMPORTANT]
> 使用本 Skill 前，必须确认项目内待处理的 `.c/.h` 文件编码为 `GB2312`（code page 936）。  
> 若文件不是 `GB2312`，执行过程中可能出现中文注释乱码、不可逆替换字符或写回失败。

用于在嵌入式 C 项目中执行统一、可复用、可验证的编码风格治理（偏 MISRA 风格），并严格控制编码、注释、括号与头文件规范。

## 功能清单

1. 项目上下文初始化  
   进入任务后先定位项目目录并检查目标路径可访问，再开始任何修改，避免误改目录。

2. 可选范围治理  
   支持只处理 `driver`、`bsp`、`app` 中的一个或多个范围，也支持全量处理；默认全量。

3. 文件筛选与边界控制  
   仅处理选定范围内的 `.c/.h` 文件，不跨目录改动，保证变更边界可控、可审阅。

4. 编码安全治理（强约束）  
   对修改文件执行严格 `GB2312` 读写（code page 936）；解码或编码失败立即停止并进入回滚流程。

5. 文件头与函数注释标准化  
   统一 `.c/.h` 顶部文件头注释格式；统一函数注释标签（如 `@brief`、`@param`、`@retval`）。

6. 括号与控制流风格统一  
   强制左大括号换行；`if/while/else if/else` 即使单行也必须使用花括号，提升可维护性与安全性。

7. 头文件与 C/C++ 兼容规范化  
   统一标准库头文件写法（`<>`）与项目头文件写法（`""`）；确保头文件具备 include guard 与 `extern "C"` 兼容块。

8. 宏与基础可读性规则收敛  
   统一宏参数括号、多语句宏 `do { ... } while (0)`、`switch-default` 等关键规则，减少歧义与隐藏风险。

9. 注释质量修正  
   清理模板化、机械化注释，改为自然工程语义描述；注释语言保持中文并聚焦模块意图。

10. 行尾与文件尾统一  
    统一 `CRLF` 换行，并确保文件以一个空行结束（`CRLF+CRLF`），降低跨工具差异问题。

11. 结果校验清单输出  
    支持通过 `rg`/字节尾检查等方式做规则验证，输出可核查结果，便于评审与验收。

12. 失败保护与回滚策略  
    出现乱码、编码异常或异常替换字符时立即停止，不继续批处理，并按备份策略恢复。

## 适用场景

- 旧项目风格不统一，需要批量对齐规范
- 交付前做风格收敛，降低评审噪音
- 含中文注释的 C 项目，需要避免编码损坏（乱码/替换字符）

## Skill 入口信息

- Skill 名称：`embedded-coding-style`
- 显示名称：`Embedded Coding Style`
- 简介：`Help with Embedded Coding Style tasks`

## 使用方式（在 Codex 中）

在任务中明确说明：

1. 项目根目录
2. 处理范围（driver / bsp / app / 组合 / 全量）
3. 是否有备份路径（建议提供）

示例指令：

```text
使用 embedded-coding-style skill，
项目根目录是 <project_root>，
只处理 driver 和 bsp，
按 SKILL.md 的规则执行并给出检查结果。
```

## 执行流程（对应 SKILL.md）

1. 初始化上下文并确认目标范围
2. 只枚举选定范围内的 `.c/.h` 文件
3. 小步编辑（文件头 -> 括号 -> 注释/签名）
4. 规范 include、C++ 兼容块与行尾
5. 执行检查清单（括号、头注释、尾字节、编码异常等）
6. 输出结果；若编码失败按回滚策略立即停止

## 关键约束

- 不可在未选定范围外修改文件
- 不允许编码自动兜底（禁止 UTF-8/UTF-16 fallback）
- 不为了风格调整改动时序/ISR/状态机行为
- 出现解码/编码异常必须立即中止并恢复

## 验证建议

- 使用 `rg` 对无花括号分支、头注释、C++ 兼容块进行扫描
- 抽查中文注释可读性，避免模板化 AI 语言
- 检查文件尾字节为 `CRLF+CRLF`（`13,10,13,10`）
