# Embedded Coding Style Codex Skill

> [!IMPORTANT]
> 使用本 Skill 前，必须确认项目内待处理的 `.c/.h` 文件编码为 `GB2312`（code page 936）。
> 若文件不是 `GB2312`，执行过程中可能出现中文注释乱码、不可逆替换字符或写回失败。

## 功能清单

1. 项目上下文初始化（必做）
   执行前先确认项目根目录可访问，并可选检查备份路径可用性。
2. 处理范围选择（必做）
   支持 `driver` / `bsp` / `app` / 组合子集 / 全量；未指定时按全量处理。
3. 边界化文件枚举
   仅枚举并处理选定范围内的 `.c/.h` 文件，禁止越界改动。
4. 严格编码预检与读写
   全流程强制 `GB2312`（936）严格解码和编码，禁止自动识别与任何 fallback。
5. 编码异常即停与回滚
   任一文件解码或编码失败立即停止，按备份恢复，不继续批处理。
6. 文件头注释标准化
   统一 `.c/.h` 顶部文件头，包含 `@file/@author/@brief/@version/@date`。
7. 函数注释标准化
   函数注释使用 `/** ... */`，统一 `@brief @param @retval` 结构。
8. 注释语言与质量治理
   注释保持中文，清理模板化/AI腔注释，改写为自然工程语义说明。
9. 控制流括号强制化
   `if/while/else if/else` 即使单行也必须加花括号。
10. 大括号风格统一
    左大括号换行，且大括号缩进与控制语句对齐。
11. `switch/case` 安全规则
    `switch` 必须包含 `default`；每个 `case` 必须显式 `break/return` 或受控跳转。
12. 比较与字面量可读性规则
    倾向显式比较；整数字面量按需添加后缀（如 `U`/`UL`）。
13. 宏安全规则
    宏参数与表达式加括号；多语句宏统一 `do { ... } while (0)`。
14. 头文件 include 规范化
    标准库头文件使用 `<>`，项目头文件使用 `""`。
15. C/C++ 兼容块规范化
    头文件必须保留 include guard，并包含且仅包含一对 `extern "C"` 兼容块。
16. 头文件内部顺序规范化
    `.h` 内部顺序统一为：`#include -> #define -> typedef -> 函数声明 -> 变量声明`。
17. 函数签名一致性治理
    结构体指针形参统一使用 `handle` 命名，并同步更新函数体内引用。
18. 可见性与健壮性收敛
    文件私有符号优先 `static`；外部可达路径中的指针使用前需空指针检查。
19. 魔法数治理
    避免裸常量，按项目风格抽取为宏或常量定义。
20. 行尾与文件尾统一
    强制 `CRLF`，并确保文件末尾为 `CRLF+CRLF`。
21. 结果验证清单输出
    基于 `rg`、尾字节检查、头文件顺序抽查等产出可核查结果。
22. 行为安全边界保护
    风格治理不改动时序、ISR、状态机等功能行为，确保仅样式收敛。

## 使用方式（在 Codex 中）

在任务中明确说明：

1. 项目根目录
2. 处理范围（driver / bsp / app / 组合 / 全量）
3. 是否有备份路径（建议提供）

示例指令：

```text
使用 embedded-coding-style skill，
项目根目录是 <project_root>，
只处理 driver 和 bsp，
按 SKILL.md 的规则执行并给出检查结果。
```

## 执行流程（对应 SKILL.md）

1. 初始化上下文并确认目标范围
2. 只枚举选定范围内的 `.c/.h` 文件
3. 小步编辑（文件头 -> 括号 -> 注释/签名）
4. 规范 include、C++ 兼容块与行尾
5. 头文件内部顺序规范化（`#include -> #define -> typedef -> 函数声明 -> 变量声明`）
6. 执行检查清单（括号、头注释、尾字节、编码异常、头文件顺序等）
7. 输出结果；若编码失败按回滚策略立即停止

## 关键约束

- 不可在未选定范围外修改文件
- 不允许编码自动兜底（禁止 UTF-8/UTF-16 fallback）
- 不为了风格调整改动时序/ISR/状态机行为
- 出现解码/编码异常必须立即中止并恢复

## 验证建议

- 使用 `rg` 对无花括号分支、头注释、C++ 兼容块进行扫描
- 抽查中文注释可读性，避免模板化 AI 语言
- 检查文件尾字节为 `CRLF+CRLF`（`13,10,13,10`）
- 抽查 `.h` 文件内部区块顺序是否符合统一规范
